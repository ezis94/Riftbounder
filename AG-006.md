# AG-006 — Chain Priority and Passing

## Commit

`AG-006: implement Chain priority and pass resolution`

## Added

- `ChainPriorityManager`
- turn-order priority transfer
- pass-history tracking
- all-player pass detection
- one-item-at-a-time Chain resolution
- priority reassignment after resolution
- Chain-close callback
- item-add notification that resets pass history
- multiplayer pass-order support
- resolution-failure recovery
- ADR 0002
- 14 focused tests

## Rules mapped

- 333.1 — the player creating the Chain receives first Priority
- 337.1.c.3 — the newest item's controller gains Priority
- 338 — Execute / actions available to the player with Priority
- 338.1.c — passing Priority
- 339 — all-player pass detection
- 340 — resolve only the newest Chain item
- 340.2–340.4 — close the Chain or return Priority to the new newest item

## Important correction

AG-006 does not implement “two passes on an empty Chain.” Chain Priority only
exists while a Chain item exists. When the Chain empties, the surrounding turn
or Showdown controller resumes through `IChainFlowObserver`.

## Deliberate boundary

This milestone does not yet implement playing a Reaction, activating an
ability, Pending-to-Finalized processing, or Showdown Focus. Those mechanics
will call `NotifyChainItemAdded` in later milestones.
