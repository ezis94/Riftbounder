# AG-009 — Target Snapshots and Deflect

## Commit

`AG-009: implement target snapshots and Deflect costs`

## Implemented

- Immutable target snapshots.
- Target requirements by kind and location.
- Selection-time validation.
- Resolution-time eligibility checks.
- Explicit distinction between:
  - whether an object can be chosen now;
  - whether a previously chosen object can still be affected.
- Deflect as a finalized play-cost modifier:
  - adds 1 Power of any Domain for each Deflect target;
  - does not make the target immune;
  - is not checked again during resolution.
- Battlefield-only target requirements.
- Targets moved to Base become ineligible for battlefield-only instructions.
- Finalized spell Chain items retain target snapshots.
- Play-card requests can carry finalized targets.
- Focused regression tests.

## Human-validated rules cases

1. A target that gains Deflect after it was chosen remains affected.
2. Deflect adds a targeting cost; it does not prevent effects.
3. A unit targeted as "a unit at a battlefield" becomes ineligible if it moves
   to Base before resolution.
4. The spell resolves as much as possible using remaining eligible targets.

## Deliberate boundary

AG-009 does not yet implement target ownership restrictions, friendly/enemy
restrictions, specified battlefield identity, player targets, or target-count
schemas. It provides the lifecycle and extension points for those rules.
