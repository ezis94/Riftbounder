# AG-012 — Ready and Exhaust Instructions

## Commit

`AG-012: implement ready and exhaust spell instructions`

## Added

- Ready/exhausted state for cards registered in `BoardState`.
- Idempotent `Ready()` and `Exhaust()` state transitions.
- Concrete `ready` instruction handler.
- Concrete `exhaust` instruction handler.
- Invalid targets are skipped through the existing partial-resolution pipeline.
- Full spell-resolution integration test.

## Semantics

- A newly registered board card defaults to ready.
- Readying an already ready card succeeds but changes nothing.
- Exhausting an already exhausted card succeeds but changes nothing.
- `AffectedTargetCount` reports actual state changes.
- A target moved to Base before resolution is skipped when the instruction
  requires a unit at a battlefield.

## Deliberate boundary

This patch does not implement Stun. Stun has additional turn-scoped rules
beyond simply exhausting a unit.

This patch also does not connect card readiness to the Awaken Phase yet. The
current Awaken implementation only readies Runes.
