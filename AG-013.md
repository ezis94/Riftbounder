# AG-013 — Canonical Locations and Zones

## Commit

`AG-013: add canonical game locations and zones`

## Added

- Stable `ZoneId` identity for every card zone.
- Nullable zone ownership so shared zones are representable.
- Canonical player-owned zones:
  - Main Deck
  - Hand
  - Base
  - Hidden
  - Trash
  - Banish
- Canonical shared locations:
  - Battlefield 1
  - Battlefield 2
- `GameBoard` as the source of truth for:
  - both player Bases;
  - exactly two shared Battlefields.
- Registration of every canonical card zone in `Game`.
- Transfers between Base, Battlefield, Hidden, Trash, and Banish use the same
  atomic zone-transfer primitive already used for Hand and Main Deck.
- Tests for zone identity, ownership, board shape, and cross-zone movement.

## Design boundary

The Chain remains represented by the existing typed `Chain` container rather
than by a general `Zone`, but `ZoneKind.Chain` is reserved for rules
classification and future unified location reporting.

This milestone does not yet implement:
- battlefield cards;
- battlefield control;
- scoring;
- movement actions or costs;
- unit controller state;
- legal play destinations.

It establishes the canonical locations those systems will use.
